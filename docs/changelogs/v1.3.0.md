# v1.3.0 — Broker DM Redesign

## Added
- `broker whoami` — print identity derived from cwd.
- `broker send --to <id>[,<id>] CONTENT` — direct-message mode on the existing `send` subcommand; auto-derives `--identity` from cwd.
- `broker broadcast CONTENT` — fan out to every registered identity.
- `broker reply-all --to-message MID CONTENT` — reply to all recipients of a prior DM, excluding self.
- `broker history [--from X] [--since ISO] [--sent]` — cursor-free peek at inbox (or outbox with `--sent`).
- `broker read` and `broker follow` — inbox-mode when called without a conversation ID.
- `scripts/broker_identity.py` — identity derivation from `package.json` or `git remote origin`.
- `scripts/broker_storage.py` — `InboxLog`, `OutboxLog`, `CursorStore`, `IdentityRegistry`.
- `scripts/broker_format.py` — line-oriented display format with content escape / unescape.
- `scripts/broker_constants.py` — `BROADCAST` sentinel and `RESERVED_IDENTITIES`.
- Server request handlers: `send_dm`, `send_broadcast`, `reply_all`, `history_inbox`, `read_inbox`.
- Reserved-identity enforcement on server `connect()` via `~/.mcp-broker/tokens/<id>.token`.
- `MCP_BROKER_SOCK` and `MCP_BROKER_STORAGE` environment variables override CLI defaults.

## Changed
- `broker send`, `broker read`, `broker follow` — positional conversation ID is now optional. When omitted, commands operate on the caller's DM inbox.
- `broker follow` with no conversation ID tails `~/.mcp-broker/inbox/<encoded-identity>.log` client-side (no server round-trip), so Claude Code's `Monitor` tool can tail the same file.
- Skill docs (`skills/broker/SKILL.md` and `docs/*.md`) rewritten for the DM model. Legacy room patterns documented as deprecated.

## Deprecated
- `broker create`, `broker join`, `broker leave` — emit stderr deprecation warnings. Functionality unchanged pending a future major release.
