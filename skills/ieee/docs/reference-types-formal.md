# IEEE Formal and Legal Reference Types

Reference formats for patents, standards, handbooks, manuals, legal citations, and government documents per the IEEE Reference Style Guide for Authors.

---

## Patents (IEEE Letter L)

**Basic Format (Print):**

`[#] J. K. Author, "Title of patent," U.S. Patent x xxx xxx, Abbrev. Month, day, year.`

`[#] J. K. Author, "Title of patent," Country Patent xxx, Abbrev. Month, day, year.`

**Basic Format (Online):**

`[#] Name of the invention, by inventor's name. (year, month day). Patent Number [Type of medium]. Available: site/path/file`

**Subtypes:** print patent (U.S.), print patent (non-U.S.), patent application, online patent

**Required Fields:** inventor/author, title or name of invention, patent number, date (month, day, year)

**Optional Fields:** country (if non-U.S.), patent application number, language note (e.g., "(in Russian)"), URL (online), medium type (online)

**Examples:**

[1] J. P. Wilkinson, "Nonlinear resonant circuit devices," U.S. Patent 3 624 125, Jul. 16, 1990.

[2] S. P. Voinigescu et al., "Direct m-ary quadrature amplitude modulation (QAM) operating in saturated power mode," U.S. Patent Appl. 20110013726A1, Jan. 20, 2011.

[3] K. Klionovski, "Broadband dual-band microstrip antenna," (in Russian), RU Patent Utility Model 167296, Dec. 27, 2016.

[4] Musical toothbrush with adjustable neck and mirror, by L. M. R. Brooks. (1992, May 19). Patent D 326 189 [Online]. Available: NEXIS Library: LEXPAT File: DESIGN

[5] Screwless clip mounted computer drive, by D. Williams. (2005, Apr. 26). U.S. Patent 6,885,550 [Online]. Available: http://patft.uspto.gov/netacgi/6,885,550

**Notes:**
- Retain or request the day of the month when referencing a patent.
- Use "issued date" if several dates are given.
- Online format inverts the order: invention name first, then "by inventor's name."
- Patent applications use "Patent Appl." followed by the application number.
- Non-U.S. patents specify the country before "Patent" (e.g., "RU Patent Utility Model").

## Standards (IEEE Letter P)

**Basic Format (Print -- with corporate author):**

`[#] Title of Standard, Standard number, Corporate author, location, date.`

**Basic Format (Print -- without corporate author):**

`[#] Title of Standard, Standard number, date.`

**Basic Format (Online):**

`[#] Title of Standard, Standard number, Corporate author, location, date. [Online]. Available: http://www.url.com`

`[#] Title of Standard, Standard number, date. [Online]. Available: http://www.url.com`

**Subtypes:** print standard (with corporate author), print standard (without corporate author), online standard

**Required Fields:** title (italicized), standard number

**Optional Fields:** corporate author, location, date, URL (online)

**Examples:**

[1] *Parameter Values for Ultra-High Definition Television Systems for Production and International Programme Exchange*, Rec. ITU-R BT.2020-2, International Telecommunications Union, Geneva, Switzerland, Oct. 2015.

[2] *IEEE Criteria for Class IE Electric Systems*, IEEE Standard 308, 1969.

[3] *Letter Symbols for Quantities*, ANSI Standard Y10.5-1968.

[4] *Frequency Response and Bias*, NERC Reliability Standard BAL-003-0.1b, May 2009. [Online]. Available: http://www.nerc.com/files/BAL-003-0_1b.pdf

**Notes:**
- The standard title is always italicized.
- Standard number formats vary by organization: "IEEE Standard 308," "ANSI Standard Y10.5-1968," "Rec. ITU-R BT.2020-2," etc.
- Corporate author and location are included when available but are not always required.
- Online standards append `[Online]. Available:` and the URL (no trailing period after URL).

## Handbooks (IEEE Letter F)

**Basic Format:**

`[#] Name of Manual/Handbook, x ed., Abbrev. Name of Co., City of Co., Abbrev. State, year, pp. xxx--xxx.`

**Subtypes:** single format (print only -- treated the same as manual print format)

**Required Fields:** handbook/manual title (italicized), publisher name, publisher city, year

**Optional Fields:** edition, state/country, page range, technical series number

**Examples:**

[1] *Transmission Systems for Communications*, 3rd ed., Western Electric Co., Winston-Salem, NC, USA, 1985, pp. 44--60.

[2] *Motorola Semiconductor Data Manual*, Motorola Semiconductor Products Inc., Phoenix, AZ, USA, 1989.

[3] *RCA Receiving Tube Manual*, Radio Corp. of America, Electronic Components and Devices, Harrison, NJ, Tech. Ser. RC-23, 1992.

**Notes:**
- Handbooks and manuals (print) share the same basic format in IEEE style.
- No author name is listed; the entry begins with the italicized title.
- Edition number uses ordinal abbreviation (e.g., "3rd ed.").
- U.S. locations include city, state abbreviation, and "USA."
- Technical series numbers, when present, appear after the location.

## Manuals (IEEE Letter I)

**Basic Format (Print):**

`[#] Name of Manual/Handbook, x ed., Abbrev. Name of Co., City of Co., Abbrev. State, Country, year, pp. xxx--xxx.`

**Basic Format (Online):**

`[#] J. K. Author (or Abbrev. Name of Co., City of Co. Abbrev. State, Country). Name of Manual/Handbook, x ed. (year). Accessed: Date. [Online]. Available: http://www.url.com`

**Subtypes:** print manual, online manual

**Required Fields (Print):** manual title (italicized), publisher name, publisher city, year

**Optional Fields (Print):** edition, state/country, page range, technical series number

**Required Fields (Online):** author or company name, manual title (italicized), year, URL

**Optional Fields (Online):** edition, access date, company location

**Examples:**

[1] *Transmission Systems for Communications*, 3rd ed., Western Electric Co., Winston-Salem, NC, USA, 1985, pp. 44--60.

[2] *Motorola Semiconductor Data Manual*, Motorola Semiconductor Products Inc., Phoenix, AZ, USA, 1989.

[3] *Microsoft Office 97 Visual Basic Programmer's Guide, Microsoft Professional Editions Series*, Microsoft, Redmond, WA, USA, 1997.

[4] L. Breiman. *Manual on Setting Up, Using, and Understanding Random Forests v4.0.* (2003). Accessed: Apr. 16, 2014. [Online]. Available: http://oz.berkeley.edu/users/breiman/Using_random_forests_v4.0.pdf

[5] M. Kuhn. *The Caret Package.* (2012). [Online]. Available: http://cran.r-project.org/web/packages/caret/caret.pdf

**Notes:**
- Print manuals follow the same format as handbooks (Section F) -- no personal author, title-first.
- Online manuals may begin with an author name or company name followed by location.
- Online format uses parenthetical year after the title and edition.
- Access date format: "Accessed: Abbrev. Month day, year."
- Company-authored online manuals include location after company name.

## Legal Citations (IEEE Letter H)

Legal citations follow a distinct format that differs from other IEEE reference types. Court case names are italicized in text.

### U.S. Supreme Court Decisions

**Basic Format:**

`[#] Case name, U.S. Reports, vol. xxx, year, p. xxx.`

**Required Fields:** case name, reporter (U.S. Reports), volume, year, page

**Optional Fields:** parenthetical noting dissent or concurrence

**Examples:**

[1] *Olmstead v. United States*, U.S. Reports, vol. 277, 1928, p. 438.

[2] *Olmstead v. United States*, U.S. Reports, vol. 277, 1928, p. 478 (Justice Brandeis, dissenting).

### Lower-Court Decisions

**Basic Format:**

`[#] Case name, Reporter name, series, vol. xxx, year, p. xxx (Court name).`

**Required Fields:** case name, reporter name, series, volume, year, page, court name (in parentheses)

**Examples:**

[1] *U.S. v. Councilman*, Federal Supplement, 2nd Series, vol. 245, 2003, p. 321 (U.S. District Court for the District of Mass.).

[2] *U.S. v. Councilman*, Federal Reporter, 3rd Series, vol. 373, 2004, p. 204 (U.S. Court of Appeals for the First Circuit).

### U.S. Laws

**Basic Format (Code citation):**

`[#] U.S. Code, Title xx, section xxx, year.`

**Basic Format (Public law -- Statutes at Large):**

`[#] Name of Act, Public Law No. xxx-xx, section xxx, U.S. Statutes at Large, vol. xxx, year, p. xxx.`

**Required Fields:** title number or act name, section number, year

**Optional Fields:** public law number, volume, page, codification note

**Examples:**

[1] U.S. Code, Title 18, section 3123(a)(1),(2), 2000 and 2002 Supplement.

[2] Uniting and Strengthening America by Providing Appropriate Tools to Intercept and Obstruct Terrorism, Public Law No. 107-56, section 209, *U.S. Statutes at Large*, vol. 115, 2001, p. 285.

[3] Omnibus Crime Control and Safe Streets Act of 1968, Public Law No. 90-351, sections 801--804, *U.S. Statutes at Large*, vol. 82, pp. 211--223 (codified as amended in US Code, Title 18, sections 2510--2522, 2000 and 2002 Supplement).

**Notes:**
- In text, italicize the names of court cases.
- Supreme Court decisions cite "U.S. Reports" as the reporter.
- Lower-court decisions include the reporter series (e.g., "2nd Series," "3rd Series") and the court name in parentheses at the end.
- U.S. law citations may reference the U.S. Code directly or the Statutes at Large with the public law number.
- Codification notes in parentheses are used when citing a statute that has been codified elsewhere.
- Section ranges use en dashes: "sections 801--804."

## U.S. Government Documents (IEEE Letter R)

**Basic Format (Print):**

`[#] Legislative body. Number of Congress, Session. (year, month day). Number of bill or resolution, Title. [Type of medium]. Available: site/path/file`

**Basic Format (Online):**

`[#] Legislative body. Number of Congress, Session. (year, month day). Number of bill or resolution, Title. [Online]. Available: site/path/file`

**Subtypes:** print government document, online government document

**Required Fields:** legislative body, congress number, session, date (year, month, day), bill/resolution number, title (italicized)

**Optional Fields:** medium type, URL or database reference

**Examples:**

[1] U.S. House. 102nd Congress, 1st Session. (1991, Jan. 11). *H. Con. Res. 1, Sense of the Congress on Approval of Military Action*. [Online]. Available: LEXIS Library: GENFED File: BILLS

**Notes:**
- The legislative body (e.g., "U.S. House," "U.S. Senate") starts the reference.
- Congress number uses ordinal form (e.g., "102nd Congress").
- Session is given as ordinal (e.g., "1st Session").
- The date in parentheses uses the format "(year, month day)."
- The bill/resolution number and title are italicized together.
- The print and online formats are essentially the same; online adds `[Online]. Available:` with the URL or database path.
- Database references (e.g., "LEXIS Library: GENFED File: BILLS") are acceptable in place of URLs.

## Cross-Reference: Field Summary by Type

| Type | Key Required Fields | Distinguishing Feature |
|------|-------------------|----------------------|
| Patent (print) | author, title, patent number, date | Author-first; "U.S. Patent" or "Country Patent" |
| Patent (online) | invention name, inventor, patent number, date, URL | Invention-name-first; "by inventor" |
| Standard (print) | title, standard number | Title-first; no author; standard org number |
| Standard (online) | title, standard number, URL | Same as print + `[Online]. Available:` |
| Handbook | title, edition, publisher, location, year | Title-first; no author; edition prominent |
| Manual (print) | title, publisher, location, year | Same format as handbook |
| Manual (online) | author/company, title, year, URL | Author/company-first; parenthetical year |
| Legal (court) | case name, reporter, volume, year, page | Case-name-first; reporter citation style |
| Legal (law) | code/act, section, year | Code or act name first |
| Government | legislative body, congress, session, date, bill number, title | Legislative-body-first; congress/session format |
